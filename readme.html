ВАЖНО: прочитайте это перед установкой

Чтобы не получить ошибку вида CANNOT MODIFY HEADER INFORMATION, не пользуйтесь стандартным БЛОКНОТОМ (для пользователей ОС Windows) для редактирования каких-либо файлов WordPress.
Особенно это касается файла wp-config.php
В WordPress используется кодировка utf-8
Если вы вносите изменения в файлы движка или файлы Темы, необходимо сохранять их в кодировке utf-8 без сигнатуры BOM
НЕ ВСЕ РЕДАКТОРЫ УМЕЮТ РАБОТАТЬ С UTF-8 БЕЗ BOM

Отключайте BOM
Воспользуйтесь лучше одним из этих редакторов:
UnicEdit,
Notepad++,
Notepad2.
При сохранении файла многие текстовые редакторы предлагают флажок «Include Unicode Signature (BOM)», «Add Byte Order Mark» или нечто подобное. Прежде всего убедитесь, что в вашем редакторе это есть. Если похожей настройки не обнаружено (как, например, в «Блокноте») — пользоваться таким редактором для серьёзных задач не стóит. Найдя этот флажок — отключите его.
Рекомендуется также почитать эту запись и пользоваться плагином поиска BOM в будущем.

Снижение потребления ресурсов

1. Вы можете существенно снизить потребление ресурсов WordPress, если отредактируете в вашем новом файле конфигурации (wp-config.php, находится в корневой папке где установлен WordPress) строку, отвечающую за работу локализации 
закомментируйте строку
define ('WPLANG', 'ru_RU'); 
и раскомментируйте эту строку
if (strpos($_SERVER['REQUEST_URI'], 'wp-admin')) define ('WPLANG', 'ru_RU'); else define ('WPLANG', 'ru_RU_lite'); 

2. Если ваши плагины или тема оформления используют вывод текстовой информации на лицевую часть сайта, то скопируйте их языковые модули с названиями *_lite.mo. Например если оригинальный файл локализации плагина называется wptuner-ru_RU.mo , то скопируйте его с названием wptuner-ru_RU_lite.mo (естественно файл лежать должен в той же папке где и обычный файл локализации плагина/темы)

3. Эффективный поиск "узких мест" в вашем блоге лучше всего делать с помощью вот этого плагина . Например вы можете найти какие плагины работают некорректно, и какие функции сколько потребляют памяти и сколько делают запросов к базе.

4. Получите от вашего блога максимальную скорость! Заблокируйте запросы на обновление движка и плагинов через плагин (описание работы по ссылке)

Установка: Знаменитая установка за 5 минут

  1. Распакуйте архив в пустую директорию.
  2. Откройте wp-config-sample.php текстовым редактором, например WordPad или аналогичным и заполните данные для подключения к базе данных.
  3. Сохраните файл как wp-config.php
  4. Загрузите все на сервер.
  5. Откройте /wp-admin/install.php в вашем браузере. Это должно настроить таблицы, необходимые для блога. Если будет ошибка, то дважды проверьте ваш файл wp-config.php и попробуйте заново. Если ошибка произойдет снова, пожалуйста обратитесь в  форум поддержки (англ) или русский форум поддержки1 и русский форум поддержки2 и соберите как можно больше данных, которые смогут пригодиться чтобы вам помочь.
  6. Пароль дается вам при установке.
  7. Скрипт установки затем должен отправить вас на страницу входа. Войдите с именем пользователя admin и паролем, сгенерированном для вас во время установки. Вы можете затем перейти в раздел 'Профиль' чтобы изменить пароль.

Обновление

Прежде чем вы что либо обновляете, будьте уверены что у вас имеются архивные копии всех файлов которые могут быть модифицированы, например index.php. Не забывайте также сделать архивную копию базы данных, например с помощью PhpMyAdmin или плагина для архивации базы данных

Обновление с любой предыдущей версии до WordPress 2.8.6:

  1. Удалите ваши старые файлы WP, сохраните то что вы изменили. Папку wp-content не трогать, в ней хранятся ваши плагины, тема оформления и загруженные файлы (если вы не меняли путь к папке загрузок)!
  2. Загрузите новые файлы.
  3. Введите в вашем браузере /wp-admin/upgrade.php. 
  4. Возможно вы ожидали большего? Это все! (в большинстве случаев)
  5. Если у вас проблемы или этой информации недостаточно, то читайте этот FAQ и ссылки в нем тоже полистайте

Системные требования

  • PHP версии 4.3 или выше. Некоторые плагины требуют обязательно PHP версии 5.0 или выше
  • MySQL версии 4.0 или выше. Корректная работа с русскими буквами гарантирована только при использовании версий 4.1 или выше

Переход с другой системы

WordPress может импортировать данные из некоторого количества систем. Для начала вы должны иметь установленный и работающий WordPress.

XML-RPC и Atom интерфейс

Вы можете писать в ваш блог WordPress с помощью инструментов Windows Live Writer, Ecto, Bloggar, Radio Userland (это означает, можно использовать функцию Radio емейл-в-блог), NewzCrawler, и другие инструменты, которые поддерживают Blogging API! :) Вы можете прочитать больше о XML-RPC поддержке в Кодексе.

Постинг через Email

Вы можете постить в блог через почтовый клиент! Для установки перейдите в экран настроек "Написание" и заполните детали подключения для вашего секретного аккаунта POP3. Затем вы должны настроить wp-mail.php выполняться периодически для проверки новых записей. Вы можете сделать это с помощью Cron-jobs, или если ваш хостинг не поддерживает это, вы можете присмотреться к некоторым сервисам мониторинга сайтов, и заставить их проверять ваш wp-mail.php URL.

Постинг очень простой: Любое электронное письмо, отправленное на адрес будет автоматически опубликовано, с темой письма в качестве заголовка. Лучше держать этот адрес в секрете. Скрипт удаляет электронные письма которые были опубликованы.
